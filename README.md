Clinic Management System – ER Diagram

This ER diagram represents a basic digital system for a clinic to manage its daily operations like appointments, consultations, diagnostics, and payments.

The system includes core entities such as Patients, Doctors, Appointments, Consultations (Visits), Diagnostic Tests, Reports, and Payments. A patient can book multiple appointments and visit the clinic multiple times. Each appointment may or may not result in a consultation.

During a consultation, a doctor can prescribe one or more diagnostic tests. These tests generate reports later, which are linked back to the consultation and patient. Doctors belong to specific specialties, and each doctor can handle many patients.

Payments are connected to appointments or consultations depending on when the billing happens.

The design keeps the system simple but scalable, focusing on real clinic workflows instead of overcomplicating it like a hospital system.
